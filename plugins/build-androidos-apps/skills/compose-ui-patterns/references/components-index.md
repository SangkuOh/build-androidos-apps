# Compose Components Index

Read the relevant official documentation before implementing a complex component.

| Component family | Review focus |
| --- | --- |
| Lazy lists and grids | Stable keys, content types, scrolling state, item cost |
| Forms and text input | Hoisted values, IME insets, validation timing, focus behavior |
| Dialogs and sheets | One presentation model, dismissal semantics, state restoration |
| Navigation | Route ownership, deep links, back behavior, adaptive panes |
| Images | Decode size, caching, content descriptions, placeholders |
| Pull-to-refresh and async content | Explicit loading state, cancellation, retry behavior |
| Adaptive scaffolds | Window size, insets, navigation suite, pane continuity |

Useful official entry points:

- Compose UI: `https://developer.android.com/develop/ui/compose`
- Lists: `https://developer.android.com/develop/ui/compose/lists`
- Text input: `https://developer.android.com/develop/ui/compose/text/user-input`
- Accessibility semantics: `https://developer.android.com/develop/ui/compose/accessibility/semantics`
- Adaptive UI: `https://developer.android.com/develop/ui/compose/building-adaptive-apps`
